## 🧩 지금 코드는 이렇게 동작합니다 (그리고 이게 문제입니다)

**이 표가 이 과제의 핵심입니다.** 아래 "문제" 칸을 해결하면 과제가 완성됩니다.

| # | 기능 | 지금 동작 | ❌ 문제 (= 당신이 고칠 것) |
|---|------|-----------|----------------------------|
| 1 | 초기 로드 | `GET /api/tasks`로 5,000개를 불러와 렌더 | 로드 실패(~2%) 시 **에러 처리·재시도 없음**. 로딩/에러/빈 상태 구분 없음 |
| 2 | 카드 드래그 | 화면(state)만 바뀜 | **서버에 저장 안 됨** → 새로고침하면 원복. 낙관적 반영·실패 롤백·경쟁 상태 전부 없음 |
| 3 | 대량 렌더 | 5,000개를 **전부** DOM에 그림 | 가상화 없음 → **스크롤·드래그가 버벅임** |
| 4 | 추가/수정/삭제 | API 함수만 있고 호출 안 함 | **CRUD 화면이 없음** |
| 5 | 검색/필터 | 없음 | 없음 |
| 6 | 테스트 | 순수 함수 예시 1개 | 핵심 로직 테스트 필요 |

**코드에서 확인할 위치**
- 문제 1·2 → `src/Board.tsx` (`useEffect` 로드, `moveTask` 함수의 `TODO(P1)` 주석)
- 문제 3 → `src/components/Column.tsx` (`tasks.map`으로 전량 렌더)
- API 함수 → `src/api/client.ts` (`getTasks/createTask/updateTask/deleteTask` — 쓰기 함수는 아직 아무도 호출하지 않음)

## ✅ 요구사항

### P1 — 필수 (여기서 실질 평가)

1. **로드 상태 처리** — 로딩 / 에러(재시도 가능) / 빈 상태를 각각 표현
2. **낙관적 업데이트 + 실패 롤백** ★핵심
   - 이동·수정·삭제·생성 시 **UI 먼저 반영** → 서버 요청
   - 실패(~15%)하면 **이전 상태로 정확히 되돌리고** 사용자에게 알림
   - (스피너 돌리고 성공해야 반영하는 방식은 미충족)
3. **경쟁 상태 처리** — 같은 카드를 빠르게 연속 이동해도 **최종 상태 = 서버 상태**. 늦게 온 응답이 최신을 덮어쓰지 않게
4. **대량 데이터 성능(5,000개)** — 검색·필터·드래그가 버벅이지 않게 (방법은 자유: 가상화·메모이제이션 등)
5. **CRUD** — 추가(제목·우선순위 필수) / 수정 / 삭제(확인 다이얼로그)
6. **핵심 로직 유닛 테스트** — 필터·정렬·롤백 등 순수 로직의 정확성 증명

### P2 — 권장 (시간이 되면)

- 409 충돌 처리 UX · 실패 요청 재시도/백오프 · 다중 탭 동기화 · 키보드 접근성 · 검색 디바운싱/다중 필터

### 🤔 일부러 정답을 안 준 것 (판단력 평가)

스스로 정하고 **`DECISIONS.md`에 근거**를 적으세요. 결정보다 **근거의 타당성**을 봅니다.

- 네트워크가 **완전히 끊기면** 어떻게 동작해야 하나?
- **409 충돌** 시 사용자에게 무엇을 보여주고 어떻게 해소하나?
- 실패한 쓰기를 **자동 재시도** vs **수동 재시도** 중 무엇으로?

### 🚫 평가하지 않음 (시간 쓰지 마세요)

다크 모드 · 화려한 애니메이션 · 정교한 비주얼 디자인

## 📦 mock API & 데이터

모든 요청에 **200~800ms 랜덤 지연**. 실패율·지연은 `src/mocks/config.ts`에서 조절.

| Method | Endpoint | 실패 |
|--------|----------|------|
| GET | `/api/tasks` | 드물게 500 |
| POST | `/api/tasks` | ~15% 500 |
| PATCH | `/api/tasks/:id` | ~15% 500, **`version` 불일치 시 409**(+서버 최신 상태) |
| DELETE | `/api/tasks/:id` | ~15% 500 |

- **낙관적 동시성**: 태스크마다 `version`(정수)이 있고, `PATCH`는 이 값을 함께 보냅니다. 서버 값과 다르면 **409**. `src/api/client.ts`의 `ApiError.payload.current`로 서버 최신 상태를 읽을 수 있습니다.
- ⚠️ **mock API를 우회해 localStorage로 대체 금지**(비동기 처리가 과제 본질). 다중 탭 동기화 등 부가 용도의 localStorage는 무방.

```ts
type Task = {
  id: string
  title: string; description?: string
  status: 'todo' | 'in-progress' | 'done'
  priority: 'high' | 'medium' | 'low'
  tags?: string[]; assignee?: string
  createdAt: string; updatedAt: string
  version: number   // 낙관적 동시성 제어용
}
```
